# Contributing Guide

## Общие принципы

- соблюдайте существующий стиль кода
- не смешивайте рефакторинг и фичи в одном PR
- один PR — одна задача

## Как внести изменения

1. Fork репозитория
2. Создайте feature-ветку
3. Внесите изменения
4. Убедитесь, что проект собирается
5. Откройте Pull Request

## Pull Request требования

- осмысленное описание
- ссылка на Issue (если есть)
- без лишних форматирований
