## Быстрый старт

1. Клонировать репозиторий:

```bash
git clone https://github.com/DanyaBooba/aquarium-mini-web-client.git
```

2. Перейти в директорию клиента:

```bash
cd aquarium-mini-web-client/client
```

3. Установить зависимости:

```bash
npm install
```

4. Запустить приложение:

```bash
npm start
```

После запуска клиент будет доступен по адресу:

```
http://localhost:3000
```

## Описание

**Аквариум мини** — open source веб‑клиент для одноимённого мессенджера. Клиент реализован на React и предназначен для кастомизации пользовательского интерфейса, экспериментов с UX, а также разработки собственных альтернативных клиентов, совместимых с API проекта.

Клиент использует публичное API мессенджера «Аквариум мини», доступное по адресу: `https://mini.aquarium.org.ru/api`

## Скриншоты

В папке `assets` размещены скриншоты интерфейса приложения. Наличие скриншотов рекомендуется для быстрого визуального ознакомления с проектом.

## Статус проекта

Проект находится в активной разработке.

Функциональность и архитектура могут изменяться, однако основные API и структура клиента считаются стабильными и пригодными для использования в собственных проектах.

---

## Цели проекта

* предоставить лёгкий и расширяемый веб-клиент для мессенджера «Аквариум мини»
* служить основой для кастомных клиентов и UI-экспериментов
* упростить разработку альтернативных интерфейсов без необходимости писать backend

```
src/
├─ components/    # Переиспользуемые UI-компоненты
```

## Требования и сборка проекта

### Требования

* Node.js 18+
* npm

### Сборка проекта (production)

Для создания production-сборки используется стандартный скрипт React.

```bash
npm run build
```

После выполнения команды:

- будет создана папка `dist/`
- в ней — оптимизированная production-версия клиента
- сборка готова к деплою на любой статический хостинг (Nginx, Vercel, Netlify и т.д.)

## Создание собственного интерфейса

Проект открыт именно для кастомизации и расширения интерфейса.

Рекомендуемый сценарий работы:

1. Сделать **fork** репозитория
2. Клонировать свой fork локально
3. Изменять UI‑компоненты в `src/components`
4. Изменять страницы в `src/pages`
5. Обновлять стили в `src/css`
6. Настраивать API и поведение клиента в `src/config`

Backend для верстки и UI‑логики не требуется. Все изменения можно разрабатывать и тестировать локально.

---

## Работа с API

* Все HTTP‑запросы инкапсулированы в `src/utils`
* Конфигурация endpoint и параметров подключения находится в `src/config`
* Клиент допускает подключение любого совместимого backend

Это позволяет использовать проект как основу для собственного сервера или тестового API.
